What makes Eleventy special?

Flexibility

Betting on JavaScript

Not a JavaScript framework

Introduction blog post

There are a lot of Static Site Generators out there. They come in different shapes and flavours. Personally I used Jekyll, Hugo, Hexo and Grav before.

But Eleventy has something, that makes it stand out to me.

From its initial design goals, these are: Flexibility, Betting on JavaScript, not a JavaScript framework. You can find Zach's thoughts on his introduction blog post. Let's look at them in detail.

Flexibility

Mix and match template engines.

Define input and output.

Or use it as one-off script.

Flexibility

You can mix and match template engines at will. Say, having Liquid because you started off with Jekyll. At one point, you want to benefit from Nunjucks. Easy with Eleventy!

Eleventy even allows you to define a template inheritance chain - between different template languages if you feel like it!

You are also free to choose input and output directory. Or even use it to transform a single file!

Betting on JavaScript

Source: Modulecounts

Betting on JavaScript

The JavaScript ecosystem is vast! I ran modulecounts against different module repositories. NPM clocks at 1.5 million in Jan 2021. That's a 1.5 and five zeroes! Followed by Maven with about 370k Java modules. That's four times as much choice!

You will likely find a library for every problem you can think of. Let's benefit from this fact.

I hear your argument: That would be true for other Static Site Generators like Gatsby, too. Which leads us to …

Not a JavaScript framework

Not a JavaScript framework

Unlike Gatsby, Eleventy does not force you to learn a JavaScript framework. It's plain Node.js. It does not recommend or enforce certain frameworks (by default). That being said, there are experiments to use it with Svelte or Vue.js.

It neither forces you to use a certain template language. More about that later.

Let's shortly recap, what makes Eleventy special.

It's easy to get started. You have the NPM behind you.

Plugins fall into three categories. Filters take some content and transform it into something else to be used in-place. Shortcodes can be thought of as macros. So a placeholder for a larger code snippet. Transforms operate on a whole file.

You can write your content in Markdown. Or plain HTML. (One of the pillars of Static Site Generators!)

Next to the default collection generated from tags you can use a file glob. Or filter an existing collection.

The async data used for directory files can be on disk. I'm curating an events list for example. Or you can fetch it over the wire. I'm pulling my tweets onto my website this way.
